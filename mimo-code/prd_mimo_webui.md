# MiMo Code Web UI — 产品需求文档 (PRD)

## 项目信息

- **Language**: 中文 (CN)
- **Programming Language**: Vite + React + MUI + Tailwind CSS（前端重构），Python server.py 代理层沿用或升级为 Hono/Node.js 适配
- **Project Name**: mimo_code_webui
- **Repository Path**: mimo-code/rootfs/usr/share/mimocode/webui/
- **原始需求**：为 MiMo Code（终端原生 AI 编码助手）的 HA Add-on 版本开发功能完整的 Web UI，通过 HTTP API 实现与 TUI 等效的交互体验。

---

## 1. 产品目标

1. **功能对等**：Web UI 覆盖 TUI 核心交互功能（模型选择、配置管理、代理模式、命令面板），实现不低于终端 UI 的功能完整度
2. **轻量易用**：保持单页应用架构，无需外部数据库，部署即用，学习成本低
3. **可配置**：支持主题切换、提供商管理、会话搜索等提高日常效率的增强功能

## 3. 需求池

### P0 — Must Have（必须实现）

| ID | 需求 | API 依赖 | 说明 |
|----|------|----------|------|
| P0-1 | **模型选择器** | `GET /provider` → 解析可用模型列表 | 聊天头部下拉框或弹出选择器，切换后后续请求使用选定模型 |
| P0-2 | **配置设置页** | `GET /config/*`, `POST /config/*` | 设置页面：provider 选择、默认 model、主题、工作目录等 |
| P0-3 | **命令面板** | `GET /command` | 类 VS Code `Ctrl+K` 命令面板：模糊搜索、执行命令、快捷键提示 |
| P0-4 | **代理模式切换** | `GET /agent` | 聊天头部显示当前代理/模式，支持 Plan / Agent / YOLO 三档切换 |
| P0-5 | **主题切换** | 读取 config 中的 theme 字段 | 暗色/亮色/高对比度等主题，CSS 变量驱动，跟随配置持久化 |
| P0-6 | **会话搜索** | 前端过滤 `GET /session` 列表 | 侧边栏顶部搜索框，实时模糊匹配会话标题 |

### P1 — Should Have（建议实现）

| ID | 需求 | API 依赖 | 说明 |
|----|------|----------|------|
| P1-1 | **提供商管理页** | `GET /provider`, `GET /provider/auth`, `POST /provider/:id/oauth/*` | 显示各提供商连接状态、OAuth 登录引导、断开连接 |
| P1-2 | **技能展示页** | `GET /skill` | 列出所有可用技能，显示名称/描述/启用状态 |
| P1-3 | **成本统计** | 从消息响应中提取 token 用量 | 会话级和全局 token 消耗统计、预估成本、图表展示 |

### P2 — Nice to Have（可选）

| ID | 需求 | 说明 |
|----|------|------|
| P2-1 | **文件浏览器** | 基于 `/file/*` API 实现内置文件树 |
| P2-2 | **Git 面板** | 基于 `/vcs/*` API 显示分支、diff、提交记录 |
| P2-3 | **实验性功能入口** | 基于 `/experimental/*` 展示实验功能开关 |
| P2-4 | **MCP 管理** | 基于 `/mcp/*` API 管理 MCP 工具 |
| P2-5 | **PTY 终端** | 基于 `/pty/*` WebSocket 实现内嵌终端 |
| P2-6 | **工作流编辑器** | 基于 `/workflows/*` API 的可视化工作流管理 |

## 4. UI 设计稿描述

### 整体布局

```
┌─────────────────────────────────────────────────────────┐
│  ┌──────────────┐  ┌──────────────────────────────────┐ │
│  │  Session Bar  │  │        Header Bar                │ │
│  │  (搜索框)     │  │  [模型选择▼] [代理模式▼] [⚙]     │ │
│  │               │  ├──────────────────────────────────┤ │
│  │  会话列表      │  │                                  │ │
│  │  ○ 会话1      │  │       聊天消息区域                │ │
│  │  ○ 会话2      │  │                                  │ │
│  │  ○ 会话3      │  │                                  │ │
│  │               │  │                                  │ │
│  │  [+ 新会话]   │  ├──────────────────────────────────┤ │
│  └──────────────┘  │  │ 输入框 [Ctrl+K 打开命令面板]    │ │
│                     │  └──────────────────────────────────┘ │
└─────────────────────────────────────────────────────────┘
```

### 4.1 侧边栏（Session Sidebar）

- **顶部**：搜索输入框，输入时实时过滤会话列表（P0-6）
- **中部**：会话列表，显示标题、时间、消息预览
- **底部**：「+ 新会话」按钮
- 当前会话高亮，右键菜单（重命名/删除）

### 4.2 聊天头部（Header Bar）

- **左侧**：当前会话标题 + 会话信息
- **右侧**：
  - **模型选择器**（P0-1）：下拉列表，数据源自 `GET /provider`，切换后 `localStorage` 记录并发送到后端
  - **代理模式选择器**（P0-4）：Plan / Agent / YOLO 三档切换，显示当前模式徽标
  - **设置按钮** ⚙：打开设置面板

### 4.3 聊天消息区域

- 流式渲染 AI 响应（现有功能增强）
- 消息气泡展示 markdown 渲染内容（代码块语法高亮）
- 消息操作：复制、重新生成
- 底部自动滚动

### 4.4 输入区域

- 多行文本输入框，支持 Shift+Enter 换行
- 发送按钮
- Ctrl+K 快捷键提示标签 → 打开命令面板

### 4.5 命令面板（P0-3）

- 模态覆盖层，居中弹出
- 搜索输入框自动聚焦
- 命令列表：图标 + 名称 + 快捷键 + 描述
- 键盘导航（↑↓ 选择，Enter 执行，Esc 关闭）
- 数据源：`GET /command` API

### 4.6 设置面板（P0-2）

- 抽屉式（Drawer）或全屏覆盖
- **Tab 分页**：
  - **General**：默认 model、工作目录、语言
  - **Provider**：各提供商切换开关（P1-1 扩展入口）
  - **Appearance**：主题选择（暗色/亮色/高对比度）
  - **Advanced**：实验性功能开关
- 修改即保存，调用 `/config/*` API

### 4.7 提供商管理页（P1-1）

- 卡片式列表展示每个提供商
- 每张卡片：Logo、名称、连接状态（已连接/未连接/错误）、操作按钮
- OAuth 流程：点击「授权」→ 新窗口打开 OAuth → 回调处理 → 刷新状态

### 4.8 技能展示页（P1-2）

- 网格或列表视图
- 每项：技能名称、描述、启用/禁用开关、标签

### 4.9 成本统计页（P1-3）

- 会话级：当前会话 token 计数 + 预估成本
- 全局：按日/周/月聚合的 token 消耗折线图
- 各 model 的消耗占比饼图

## 5. 待确认问题 (Open Questions)

| # | 问题 | 建议方向 |
|---|------|---------|
| Q1 | 模型选择切换后，是立即影响整个应用还是仅对新会话生效？ | 建议仅对新消息生效，当前会话保持原有模型 |
| Q2 | 命令面板中的命令是否允许带参数输入？ | 建议简单命令直接执行，复杂命令弹出参数表单 |
| Q3 | 主题配置存放在远端 `/config/*` 还是本地 localStorage？ | 建议两端同步 — 配置写回 `/config/*`，本地作缓存 |
| Q4 | 代理模式切换是否需要重新创建会话？ | 建议模式切换是会话级的，切换后后续消息使用新模式 |
| Q5 | 会话搜索是前端过滤还是后端 API 搜索？ | `/session` API 若不支持搜索参数，先做前端过滤，后续可加后端 |
| Q6 | 流式响应当前是通过 POST 同步等待还是 SSE？ | 确认 `POST /session/:id/message` 是否支持 SSE 流式返回，若支持则实现打字机效果 |
| Q7 | OAuth 流程在 Web UI 中的处理方式 — 弹出新窗口还是当前页跳转？ | 建议弹出新窗口，回调后自动关闭并刷新状态 |
| Q8 | 成本统计的数据来源 — 从消息响应中解析还是独立 API？ | 确认是否有独立的 `/cost` 或 `/stats` 端点；若无则从消息元数据中提取 |
