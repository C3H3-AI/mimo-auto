# MiMo Code Web UI 产品需求文档

## 产品目标
1. **功能对等** — Web UI 覆盖 TUI 核心交互（模型选择、配置管理、代理模式、命令面板）
2. **轻量易用** — 保持 SPA 架构，部署即用
3. **可配置** — 主题切换、提供商管理、会话搜索等增强功能

## 需求优先级

### P0（6项，必须实现）
| ID | 需求 | API |
|----|------|-----|
| P0-1 | 模型选择器 | `GET /provider` |
| P0-2 | 配置设置页 | `GET/POST /config/*` |
| P0-3 | 命令面板 (Ctrl+K) | `GET /command` |
| P0-4 | 代理模式切换 (Plan/Agent/YOLO) | `GET /agent` |
| P0-5 | 主题切换 | config theme 字段 |
| P0-6 | 会话搜索 | 前端过滤 session 列表 |

### P1（3项，增强功能）
| ID | 需求 | API |
|----|------|-----|
| P1-1 | 提供商管理 + OAuth | `/provider/auth`, `/provider/:id/oauth/*` |
| P1-2 | 技能展示 | `GET /skill` |
| P1-3 | 成本统计 | 从消息元数据提取 |

### P2（6项，可选）
文件浏览器、Git面板、实验功能入口、MCP管理、PTY终端、工作流编辑器

## UI 设计
1. 左侧：会话列表（含搜索框）
2. 聊天头部：模型选择器 + 代理模式切换
3. 聊天消息区：流式渲染
4. 输入区：支持 Ctrl+K 命令面板
5. 命令面板：模态覆盖层，模糊搜索+键盘导航
6. 设置面板：抽屉式，4个Tab分页
7. 提供商管理：卡片列表+OAuth流程
8. 技能展示：网格/列表视图
9. 成本统计：图表展示
